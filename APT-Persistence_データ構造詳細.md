## 1. 全体の構造：「1台のVM = 1フォルダ」

このデータセットの基本単位は **「1台の仮想マシン（VM）から採取したログ一式」** である。

配布は2つのZIPに分かれている：

```
apt-persistence/Datasets/
├── C_Data.zip  （546 MB）  ← 正常マシン 112台分
└── I_Data.zip  （278 MB）  ← 攻撃注入済みマシン 26台分
```

ZIPを展開すると、マシン番号（整数）ごとにフォルダが並ぶ：

```
C_Data/
├── 1/       ← VMその1
├── 2/       ← VMその2
├── 3/
│   ...
└── 112/     ← VMその112

I_Data/
├── 1/
├── 2/
│   ...
└── 26/
```

## 2. 1マシンフォルダの中身（共通構造）

```
{マシン番号}/
├── Description/
│   ├── Description.yml           ← OSバージョン＋インストール済みソフト一覧
│   ├── install_choco_software.ps1 ← Chocolateyインストールスクリプト（一部マシン）
│   └── GT.yml                    ← 【I_Dataのみ】実施した攻撃の詳細
├── Evtx_Logs/
│   ├── Application.evtx
│   ├── Security.evtx
│   ├── Sysmon.evtx
│   ├── System.evtx
│   └── TaskScheduler.evtx
└── Wazuh-Alerts/
    └── alerts.json
```

### 各ファイルの内容

| ファイル | 形式 | 内容 | サイズ感 |
|---------|------|------|---------|
| `Description.yml` | YAML | そのVMのOS名・バージョン＋インストール済みアプリ全リスト | 1〜15KB |
| `install_choco_software.ps1` | PowerShell | Chocolateyでインストールを実行したスクリプト | 2〜3KB |
| `GT.yml` | YAML（フォーマットは統一されていない） | 実施した攻撃手法（MITRE ID）＋実際のコマンド | 0.1〜数十KB |
| `Application.evtx` | Windowsイベントログ | アプリケーション層のイベント | 約1.1MB（ほぼ全マシン共通） |
| `Security.evtx` | Windowsイベントログ | 認証・プロセス追跡・オブジェクトアクセス等 | **1〜261 MB（マシンによって大きく異なる）** |
| `Sysmon.evtx` | Windowsイベントログ | プロセス生成・ネットワーク接続・ファイル操作等 | **4〜398 MB（マシンによって大きく異なる）** |
| `System.evtx` | Windowsイベントログ | システムレベルのイベント（起動・ドライバ等） | 約1.1MB（ほぼ全マシン共通） |
| `TaskScheduler.evtx` | Windowsイベントログ | スケジュールタスクの実行ログ | 1〜6 MB（感染マシンで増大） |
| `alerts.json` | JSON | Wazuh SIEMが生成した補助アラート | 2〜354 MB |

## 6. GT.yml について（重要：フォーマットが統一されていない）

GT.ymlは感染マシンごとに **フォーマットが異なる**。主に3〜4種類のスタイルが混在している。

### スタイルA：YAML、単一手法（マシン1など）

```yaml
attack_technique: T1053.005
display_name: 'Scheduled Task/Job: Scheduled Task'
atomic_tests:
  - name: Scheduled Task Startup Script
    executor:
      command: |
        schtasks /create /tn "T1053_005_OnLogon" /sc onlogon /tr "cmd.exe /c calc.exe"
```

### スタイルB：YAML、複数手法リスト（マシン2など）

```yaml
Back_door: gkupdate.exe
attack_techniques:
  - id: T1562.001
    name: Impair Defenses: Disable or Modify Tools
    tests:
      - executor:
          command: schtasks /delete /tn "Windows Defender Scheduled Scan" /f
  - id: T1053.005
    name: Scheduled Task/Job
    tests: ...
  - id: T1505.004
    name: IIS Components
    tests: ...
```

### スタイルC：YAML、シンプルなIDリスト（マシン14など）

```yaml
Atomic_red_team_techniques:
  - T1574.001: DLL Search Order Hijacking
  - T1574.006: Dynamic Linker Hijacking
  - T1574.011: Services Registry Permissions Weakness
  - T1546.008: Accessibility Features
  - T1547.003: Time Providers
```

### スタイルD：Pythonリスト形式（マシン21など）

```python
GT_21 = [
    r'"cmd.exe" /c schtasks /create /tn "T1053_005_OnLogon" ...',
    r'schtasks /create /tn "T1053_005_OnStartup" ...',
    ...
]
GTS_21 = [
    r'T1053_005_OnLogon',
    r'AtomicTask',
    ...
]
```

**実用上の意味**: GT.ymlを機械的に処理するにはマシンごとにパーサーを分ける必要がある。ただし人間が読む分には十分な情報量がある。

## 8. 正常行動の記録期間（実測）

データ収集方式は `wevtutil epl` による**スナップショット**（そのときのログバッファをまるごと取り出す）。  
「N日間連続録画」ではなく「VMが稼働していた期間のログがそのまま入っている」。  
そのためマシンごとに期間が異なる。

| マシン | 期間（実測） | Security件数 | 収集日 | 備考 |
|--------|------------|------------|------|------|
| C_Data/3 | **約1時間** | 少 | 2024-08-17 | — |
| C_Data/4 | **約14時間** | 26,071件 | 2024-09-04 | 件数が多い（FW系イベントが多い可能性） |
| C_Data/5 | **約1日** | — | 2024-09-03〜09-04 | — |
| C_Data/6 | **約21時間** | — | 2024-09-04〜09-05 | — |
| C_Data/29 | **約8分** | 29,001件 | 2024-09-09 | ⚠️ 極端に短いが件数が多い |
| C_Data/30 | **約4日** | 1,477件 | 2024-09-06〜09-09 | 期間が長く使いやすい |
| I_Data/1 | **約1.5時間** | — | 2024-09-03 | — |
| I_Data/10 | **約7分** | 28,932件 | 2024-09-23 | ⚠️ 極端に短いが件数が多い |

**重要な観察**: C_Data/29（8分・29,001件）・I_Data/10（7分・28,932件）のように、**極短時間に大量イベント**が記録されているマシンが存在する。これはWindowsフィルタリングプラットフォーム系イベント（EID 5447/5156/5158）が大量発生しているためと考えられる（感染マシスのI_Data/1でEID 5447が5,689件あったのと同様のパターン）。

全体的な収集時期は **2024年8月〜9月（一部 2024-09-23）**。  
正常マシンは **数分〜4日** と幅が広い。件数ではなく**期間の長さ**が正常行動の多様性確保に重要。

## 9. データ取得方法の要点

正常行動の「リアリティ」は以下2つで担保されている：

1. **多様なソフトウェア構成** — 7職種×10〜15本のアプリ（150種プールから選択）
2. **GHOSTSフレームワーク** — CMU SEI製の「NPCユーザー」シミュレーター。ブラウジング・ファイル操作等をVM上で自動実行し、実際のユーザー操作に近いイベントを発生させる

## 10. 利用にあたっての注意点まとめ

| 項目 | 内容 |
|------|------|
| 展開後の総サイズ | 約 **10.7 GB**（C_Data 7.6GB + I_Data 3.0GB） |
| 展開方法 | PowerShell `Expand-Archive` で可（ZIP形式） |
| まず試すなら | C_Data/30（Security 2MB・Sysmon 29MB・期間4日・Description.ymlあり） |
| GT.ymlの注意 | フォーマットが統一されていないため機械処理には注意 |

| 攻撃と正常の対応 | 同一マシン番号でも C_Data と I_Data は **別VMで独立**している |
