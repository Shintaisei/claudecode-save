# APT-Persistence 研究資料目次

このフォルダは、APT-Persistence を用いた正常行動復元研究のドキュメントを読む順番に並べたもの。

元データセット本体は `../apt-persistence/` に置いたままにしている。分析で作成したCSVは `../analysis_data/` に整理した。

## 読む順番

| 順番 | ファイル | 位置づけ |
| --- | --- | --- |
| 1 | `01_研究方針まとめ.md` | 研究背景、目的、データセット選定方針の全体像 |
| 2 | `02_データセット調査_評価.md` | ATLAS / COMISET / apt-persistence などの比較検討 |
| 3 | `03_ホスト選定分析.md` | apt-persistence の正常ホストから C_Data/96 を選んだ根拠 |
| 4 | `04_実験計画.md` | この先の実験手順、Hayabusa/Wazuh の使い分け、やること |
| 5 | `05_C_Data96_ホスト行動タイムライン整理.md` | C_Data/96 でいつ何が起きたかを時間帯別に整理 |
| 6 | `06_C_Data96_正常行動_起点アラート分類.md` | Wazuhアラートを起点にした正常行動候補の分類 |
| 参考 | `90_データセット探索プロンプト.md` | データセット調査時に使った探索プロンプト |

## 研究の流れ

1. インシデント分析自動化の目的を整理する。
2. Hayabusa / Wazuh に入力しやすい EVTX 形式のデータセットを探す。
3. apt-persistence の正常ホストから、ログ量・継続時間・行動復元しやすさで対象ホストを選ぶ。
4. C_Data/96 を対象として、まずホスト全体のタイムラインを把握する。
5. Wazuh アラートを起点に、正常行動として説明できるケースを作る。

## いま見るべき資料

現在の作業の中心は以下の2つ。

- `05_C_Data96_ホスト行動タイムライン整理.md`
- `06_C_Data96_正常行動_起点アラート分類.md`

まず `05` で C_Data/96 の時間帯ごとの行動をつかみ、その後 `06` で起点アラートごとの正常行動仮説を見る。
